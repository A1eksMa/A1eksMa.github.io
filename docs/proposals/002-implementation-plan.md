# План реализации интерактивного портфолио

## Этап 1: Фундамент и сфера "Анализ данных"

**Цель:** Быстро создать минимально жизнеспособный продукт (MVP), сфокусированный на самой сильной области.

**Задачи:**

1.  **Техническая настройка:**
    *   Инициализировать проект с помощью генератора `Hugo`.
    *   Создать базовые HTML-шаблоны для 4-х разделов (`skills`, `education`, `projects`, `contact`).
    *   Создать шапку сайта с неактивными элементами фильтров и таймлайна.

2.  **Наполнение контентом (только "Анализ данных"):**
    *   **Skills:** Создать Markdown-файлы для всех навыков из "зелёной зоны" `competency_map.md` (`Python`, `Pandas`, `SQL`, `Docker`, `FastAPI` и т.д.).
    *   **Education:** Описать 1-2 релевантных курса или книги.
    *   **Projects:** Детально описать 1-2 проекта, демонстрирующих навыки анализа данных и инженерии.
    *   **Contact:** Заполнить страницу с контактами.

3.  **Реализация MVP интерактивности:**
    *   Написать JavaScript, который реализует фильтрацию по тегам технологий на всех страницах.
    *   Реализовать базовый таймлайн-слайдер, фильтрующий контент по дате.

**Результат:** Работающий сайт, который эффективно представляет вас как специалиста по данным.

---

## Этап 2: Расширение на "Разработку" и "DevOps"

**Цель:** Добавить контент для оставшихся сфер и улучшить систему фильтрации.

**Задачи:**

1.  **Наполнение контентом:**
    *   **Skills:** Создать файлы для навыков из "жёлтой зоны" и других релевантных технологий (`Java`, `Spring`, `Go`, `Rust`, `Airflow`).
    *   **Education/Projects:** Добавить контент, связанный с разработкой и DevOps. Обновить существующие проекты, добавив в них теги и описания для новых сфер.

2.  **Улучшение фильтров:**
    *   Активировать фильтр по "Сферам" (`Анализ данных`, `Разработка`, `DevOps`).
    *   Реализовать логику "акцентов" в описаниях проектов: при выборе сферы подсвечивается релевантная часть текста.

**Результат:** Полноценное портфолио, демонстрирующее весь спектр ваших компетенций.
