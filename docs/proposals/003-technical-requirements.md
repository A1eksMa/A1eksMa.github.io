# Технические требования

## 1. Стек технологий (Technology Stack)

*   **Генератор статических сайтов:** [Hugo](https://gohugo.io/) (Extended version для поддержки SCSS).
*   **Фреймворк-надстройка:** Сборка будет производиться с использованием кастомного фреймворка [info-tech-io/hugo-templates](https://github.com/info-tech-io/hugo-templates).
*   **Интерактивность:** Клиентский JavaScript (Vanilla JS) для реализации логики фильтров, таймлайна и динамических акцентов в контенте.
*   **Стили:** SCSS (Sass) для удобной и структурированной разработки стилей.

## 2. Сборка и развертывание (Build & Deployment)

*   **Платформа CI/CD:** GitHub Actions.
*   **Процесс:**
    1.  Рабочий процесс (workflow) запускается автоматически при каждом `push` в ветку `main`.
    2.  В среде GitHub Actions происходит сборка сайта командой `hugo`.
    3.  Собранные статические файлы (из папки `public`) развертываются на GitHub Pages.
*   **Источник публикации:** В настройках репозитория (`Settings -> Pages -> Build and deployment`) источником должен быть выбран **GitHub Actions**.

## 3. Структура и контент

*   **Тип контента:** Сайт должен быть на 100% статическим (HTML, CSS, JS), без серверной части.
*   **Поддержка мультиязычности:** Проект должен быть настроен для поддержки нескольких языков (например, русского и английского) с использованием встроенных возможностей Hugo. Структура контента должна это учитывать (например, `content/ru/...` и `content/en/...`).

## 4. Тема и дизайн

*   **Тема оформления:** Будет использоваться тема [Compose](https://github.com/onweru/compose), адаптированная для работы в рамках фреймворка `info-tech-io/hugo-templates`.
