# Структура сайта: Интерактивное портфолио

## 1. Общая концепция

Сайт представляет собой интерактивную "карту компетенций", состоящую из четырех основных разделов. Управление отображением контента осуществляется через глобальную панель фильтров и таймлайн, расположенные в шапке сайта.

## 2. Глобальные элементы управления

### 2.1. Панель фильтров

*   **Фильтр по Сферам:** Ключевой фильтр, определяющий общую картину. Значения: `Анализ данных`, `Разработка`, `DevOps`.
*   **Фильтр по Технологиям:** Поле с автодополнением для выбора конкретных навыков (например, `Python`, `Docker`, `PyTorch`). Позволяет выбрать несколько значений.

### 2.2. Временная шкала (Timeline)

Это не просто фильтр, а интерактивный инструмент для визуализации профессионального пути.

*   **Режим "Прошлое" (Past Track):**
    *   Показывает освоенные навыки и реализованные проекты в выбранном временном диапазоне.
    *   Может визуализировать "глубину" погружения в технологию в определенный период (например, по количеству проектов с этим тегом).
*   **Режим "Будущее" (Future Roadmap):**
    *   При переключении на этот режим или при перемещении слайдера за пределы "сегодняшнего дня", на шкале отображаются компетенции из "красной зоны" (`Kubernetes`, `PyTorch`, `NLP` и т.д.).
    *   Это наглядно демонстрирует ваш пла�� развития и амбиции.

## 3. Структура разделов

### 3.1. Skills (Навыки)

*   **Представление:** Сетка из карточек. Каждая карточка — отдельный навык.
*   **Содержимое карточки:** Иконка технологии, название. Цветовая индикация в соответствии с картой компетенций (🟢/🟡/🔴).
*   **Примеры контента (из `competency_map.md`):**
    *   `content/skills/python.md` (🟢)
    *   `content/skills/spark.md` (🟡)
    *   `content/skills/pytorch.md` (🔴)

### 3.2. Education (Образование)

*   **Представление:** Вертикальный список или карточки.
*   **Содержимое карточки:** Название курса/книги, краткое описание, дата изучения, теги технологий.
*   **Примеры контента:**
    *   `content/education/book-design-patterns.md`
    *   `content/education/course-deep-learning-ai.md`

### 3.3. Projects (Проекты)

*   **Представление:** Крупные карточки с ключевой информацией.
*   **Содержимое карточки:** Название, краткое описание, стек технологий (теги), дата.
*   **Интерактивная особенность:** Описание проекта может содержать скрытые блоки, которые становятся видимыми при выборе соответствующего фильтра "Сферы".
    *   Например, в проекте `customer-segmentation.md` будет блок `<div data-domain="data-analysis">...</div>` с описанием аналитических методов и блок `<div data-domain="devops">...</div>` с описанием того, как сервис был развернут в Docker.

### 3.4. Contact Me (Контакты)

*   **Представление:** Статичная страница.
