## Карта компетенций `/stack/`

Центральный и самый сложный раздел. Единый источник данных — `data/competency.json`.

### Параметрическое резюме

Поведение страницы определяется URL-параметром `?role=`:

```
/stack/                  нейтральный вид — всё
/stack/?role=data-analyst
/stack/?role=ml-engineer
/stack/?role=backend
/stack/?role=devops
```

При активной "линзе":
- Timeline подсвечивает релевантные технологии, нерелевантные затухают
- Resume-блок показывает только профильные навыки, курсы, книги, сертификаты
- Переключатель ролей всегда виден — можно переключиться

### Timeline-визуализация

- Ось X: время
- Ось Y: уровень владения
- Каждая технология — отдельная линия
- Показывает рост, плато, спад (например, PHP — дорос до уровня и остановился)
- Фильтры: по языку, направлению, статусу (активный / архив)
- Реализация: Chart.js

### Resume-блок (под timeline)

Три колонки под активной линзой:
- Ключевые навыки (из competency.json, отфильтровано по роли)
- Релевантные проекты (ссылки на /projects/)
- Что изучаю (курсы, книги, сертификаты — тематические)

### Тематическое образование

Курсы, сертификаты, достижения на платформах (LeetCode и др.), книги — всё привязано к направлениям и отображается в контексте соответствующей роли. Даёт ответ на вопрос: "как давно занимался и на какой глубине".

## Проекты `/projects/`

Карточки проектов. Наполняются постепенно.

Каждый проект отвечает на вопрос: *какую задачу это решает?* — не список технологий, а бизнес-контекст.

Текущие проекты (в разной степени готовности):
- **nier** — MLP с нуля на Python/NumPy, без ML-фреймворков
- **groq** — FastAPI-обёртка над Groq API (требует рефакторинга безопасности)
- **jupyter-cloud** — self-hosted Jupyter + IDE (локально, будет опубликован)
- **Telegram-бот** — практическое применение groq (локально)
- **Data notebooks** — анализ данных с визуализациями (учебные, нужно оформить)

## Данные

```
data/
└── competency.json    ← единый источник правды для /stack/
```

Структура `competency.json` — проектируется отдельно. Будет содержать: технологию, направление, временну́ю динамику уровня владения, статус (активный/архив), привязку к ролям.

Тот же файл используется как источник для Jupyter notebook'а с визуализациями (отдельный проект в портфолио).
